Docking and Prep

Local runs prepare ligands and dock on this Mac. SLURM export prepares ligands locally, then writes cluster scripts for array docking.
 
Mac recommendation: start with 20-100 ligands, exhaustiveness 1 for smoke tests, 4-8 for small careful screens, 1-2 docking workers, and 2-8 ligand-prep workers depending on memory. HPC recommendation: use SLURM export, array jobs, exhaustiveness 4-8 for broad first-pass screens and 8-16 for prioritized rescoring, with concurrency set by cluster policy.

SLURM Export

Use this when docking thousands to millions of ligands on a cluster. GPU reservation is recorded for schedulers, but AutoDock Vina remains CPU-based unless you swap in a GPU-capable docking engine later.

Help

About OSLabLambda

OSLabLambda is open-source software created by Hyung L. Kim using open-source scientific programs and AI-assisted coding. It organizes maintained community tools into a practical drug-discovery workflow; each upstream tool retains its own license, authorship, documentation, and citation requirements.

Overall Workflow

OSLabLambda is organized as a block workflow: Docking → Hit Refinement → MD and Optimization → FEP. Each block produces files and reports that can be called up by the next block for additional simulation. The Reports tab is where completed results, ligand tables, structures, logs, and report files are reviewed.
